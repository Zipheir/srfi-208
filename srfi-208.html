208: NaN procedures

by Emmanuel Medernach (concept), John Cowan (editor), Wolfgang Corcoran-Mathe (implementation)

Status

Abstract

This SRFI provides procedures that dissect NaN (Not a Number) inexact values.

Issues

None at present.

Rationale

IEEE floating-point numbers have a set of values known as NaNs, standing for Not A Number. They are generated by such operations as (/ 0.0 0.0), the result of which could be any number whatsoever, and by (flsqrt 0.0) from SRFI 144, the result of which cannot be any number. Scheme implementations that conform to R6RS or R7RS use the external representations +nan.0 and -nan.0 for NaNs.

In fact, however, there are 252 - 1 possible NaN values, assuming the representation is a binary64 float. This SRFI makes it possible to dissect a NaN to see which of these internal representations it corresponds to.

Specification

(nan-negative? nan)

Returns #t if the sign bit of nan is 1 and #f otherwise.

(nan-quiet? nan)

Returns #t if nan is a quiet NaN, and #f otherwise. Quiet NaNs are passed through floating-point operations, whereas non-quiet (or signaling) NaNs may signal a hardware exception, depending on how the CPU's floating-point instructions have been configured.

(nan-payload nan)

Returns the remaining 51 bits of nan as an positive exact integer.

(nan= nan1 nan2)

Returns #t if nan1 and nan2 have the same sign, quiet bit, and payload; and #f otherwise.
